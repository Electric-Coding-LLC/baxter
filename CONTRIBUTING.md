# Contributing

## Branches
- main is always stable.
- Use codex/<topic> for Codex work.
- Use feature/<topic> or fix/<topic> for manual work.

## Commits
- Keep commits small and focused.
- Use imperative commit messages, e.g. "Add config parsing".

## Pull Requests
- Open a PR for non-trivial changes.
- Squash merge unless you want to preserve individual commit history.

## Checks
- go test ./...
